# Skill：面试复盘

> 面试结束后，结构化提取问题、评估表现、生成改进计划。

## 触发条件
木安提到"刚面完"、"面试复盘"、"面试被问到 X 不知道怎么答"。

## 核心原则
1. **诚实评估**——不安慰"你已经很好了"，直接说哪里丢分
2. **可改进的才记**——纯运气或面试官个人偏好的问题不纠结
3. **下一场能用**——每次复盘至少产出 1 个可复用的改进

## 执行流程

### 1. 信息收集（问木安）
- 什么岗位？几面？面试官角色？
- 被问了哪些问题？（尽量回忆完整）
- 哪些答得好？哪些卡壳了？
- 面试官的反应 / 追问方向？
- 你自己感觉如何？预估结果？

### 2. 逐题评估
对每个问题做分级：
- ✅ 答得好：说清楚了，面试官认可
- ⚠️ 及格但有优化空间：答了但不精准 / 缺结构 / 漏关键点
- ❌ 明显丢分：卡壳 / 答偏 / 暴露短板

### 3. 改进方案
对每个 ⚠️ 和 ❌ 的问题：
- 这个问题的最佳回答框架是什么？
- 木安缺的是信息、结构、还是表达？
- 给出一个改进版回答（木安能记住和复用的）

### 4. 输出格式
```
## 面试复盘：[公司] [岗位] [第几面]
日期：YYYY-MM-DD
整体评估：X/10

### 问题逐条分析
| # | 问题 | 评级 | 你的回答摘要 | 改进版 |
|---|------|------|-------------|--------|

### 关键发现
- 答得好的模式：
- 反复丢分的模式：

### 下次面试前必做
1. [具体行动，不泛泛]
2. 
3. 
```

## 校准规则
- 木安说"感觉还行"时追问具体：哪个问题让你觉得还行？面试官什么反应？
- 不帮木安合理化明显的失误。丢分就是丢分，关键是下次怎么办。
- 如果同一个短板连续出现两次，升级为"必须解决"级别，写入当前任务版。
