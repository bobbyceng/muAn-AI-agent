# Skill：拆解 JD

> 用于分析目标岗位的职位描述，判断匹配度，找出叙事缺口。

## 触发条件
木安给出一个 JD 链接或文本，或提到"这个岗位怎么样"。

## 执行流程

### 1. 拆解 JD 结构
把 JD 拆成四层：
- **硬性门槛**：学历、年限、语言、证书等不可谈判项
- **核心能力**：岗位最看重的 3-5 项能力
- **加分项**：有则加分、没有不致命的能力
- **隐含要求**：JD 没写但从岗位性质可推断的要求（如跨部门沟通、自驱力）

### 2. 匹配度判断
对每项要求做三级判断：
- ✅ 有证据支撑（能讲出具体经历）
- ⚠️ 有相关经验但需要包装角度（需要校准叙事）
- ❌ 缺乏（需要判断是否致命）

### 3. 叙事缺口分析
- 匹配度低于 60% 的岗位直接说"不建议投"，不浪费时间
- 60-80% 的岗位指出需要补的 1-2 个关键缺口
- 80%+ 的岗位给出面试准备重点

### 4. 输出格式
```
## JD 拆解：[岗位名称] @ [公司]
匹配度：X%（主观判断）
建议：投 / 谨慎投 / 不建议

### 核心匹配
| 要求 | 匹配度 | 证据/缺口 |
|------|--------|-----------|

### 叙事建议
- 面试时重点讲：
- 需要准备的问题：
- 需要回避的坑：
```

## 校准规则
- 不帮木安高估匹配度。"沾边"不等于"匹配"。
- 木安的非标准背景是事实，不回避，但要找到转化角度。
- 如果 JD 要求的核心能力木安完全没有，直说。
